# 今！やることリスト

締切までにやりたいことを登録しておいて、**時間が空いたときに「今やるとよさそうなこと」を提案してくれる**タスク管理アプリです。

「やることは分かっているのに、30分空いた瞬間に何をすればいいか思い出せない」を解消するために作りました。

## できること

- **登録** — やること・締切・「1ページ何分 × 何ページ」のような見積もりを入れて記録
- **一覧** — カレンダーに締切を並べ、その日の残り作業時間もひと目で分かる（週は月曜はじまり）
- **提案** — 空き時間（例：30分）と気分（だるい／ふつう／やる気ある）を選ぶと、今やるとよさそうな順に並べて理由つきで出す
- **自動割り当て** — 「木曜の19:00〜21:00が空いている」と登録しておくと、締切に間に合うようタスクを時間割にして詰める。間に合わないものは警告する
- **タイマー** — 提案や時間割から直接タイマーを開始し、終わったら「2ページ進んだ」を記録
- **バックアップ** — JSONで書き出し・読み込み
- オフラインでも動く PWA。ホーム画面に追加すればアプリのように使えます

## 提案のしくみ

3つの観点をそれぞれ100点満点にして、重みをつけて合計しています（`js/suggest.js`）。

| 観点 | 重み | 見ているもの |
| --- | --- | --- |
| 締切 | 0.38 | 期限までの残り日数と、「1日あたり何分やらないと終わらないか」という切迫感 |
| 時間 | 0.30 | その空き時間に収まるか、無駄なく使い切れるか（終わり切るなら加点） |
| 気分 | 0.32 | 今の集中力とタスクが要求する集中度、選んだ「やりたい種類」との一致 |

そのため、**大きなタスクは「ページ単位」で登録するほど提案の精度が上がります**。15分しか空いていなくても「3ページ進める」と提案できるからです。

## 自動割り当てのしくみ

「予定」画面で空き時間を登録して割り当てを実行すると、**締切が早いものから順に、入る空き時間へ詰めます**（`js/scheduler.js`）。締切を守ることが目的なら、この単純な規則が理論上いちばん強い方法です（最早締切優先／EDF）。

- 締切より後の日には置きません
- 「タスク全体」で登録したものは、まとめて入る枠がなければ置きません
- ページ・個数単位のものは、枠に入るぶんだけ分割して詰めます
- 入りきらなかったものは「あと何分足りないか」を添えて警告します

### AIを使う場合（任意）

設定でAPIキーを入れると、Claude に**着手する順番だけ**を相談します。

**なぜ順番だけなのか。** 「何分の枠に何ページ入るか」は算数で、アルゴリズムが必ず正確に解けます。AIに数えさせると間違える余地が増えるだけで、精度は下がります。一方で「タスク名から察するに、これは頭が冴えている時間向き」「この2つは続けてやると楽」といった判断はアルゴリズムには書けません。そこだけをAIに任せ、**時間割の計算は必ずアプリがやり直します**。

そのため、AIが存在しないタスクを返しても、順番を取りこぼしても、締切を無視した順番を返しても、出てくる時間割が壊れることはありません（AIが失敗すれば締切順に自動で切り替わります）。

- モデルは Claude Opus 4.8、応答は JSON Schema で形を固定しています（`js/ai.js`）
- APIキーは**この端末のブラウザにだけ**保存され、そこから直接 api.anthropic.com へ送られます。中継サーバーはありません
- **バックアップファイルにキーは含めません**
- 共用のパソコンでは使わないでください
- **キーは未入力でも全機能が普通に使えます**。他の人に渡すときも、その人がキーを用意する必要はありません

## データの扱い

タスクは**使っている端末のブラウザ（localStorage）にだけ**保存されます。サーバーには何も送信していないので、同じURLを他の人が開いても、その人には自分専用の空のリストが表示されます。ログインは不要です。

その代わり、ブラウザのデータを消すとタスクも消えます。大事なものは設定画面からバックアップしてください。

## 手元で動かす

ES モジュールと Service Worker は `file://` では動かないので、簡易サーバーを使ってください。

```bash
node scripts/serve.mjs      # → http://localhost:5173
```

アイコンPNGを作り直したいときは:

```bash
node scripts/make-icons.mjs
```

ビルドは不要です。必要なのは Node だけ（サーバーとアイコン生成に使うだけで、アプリ本体は素の HTML/CSS/JS）。

## GitHub Pages で公開する

1. GitHub で空のリポジトリを作る（例：`now-todo`）
2. このフォルダから push する

   ```bash
   git remote add origin https://github.com/<ユーザー名>/now-todo.git
   git push -u origin main
   ```

3. リポジトリの **Settings → Pages** を開き、Source を **Deploy from a branch**、Branch を **main / (root)** にして Save
4. 数分後に `https://<ユーザー名>.github.io/now-todo/` で開けます

パスはすべて相対パスなので、サブディレクトリ配信のままで動きます。スマホで開いて「ホーム画面に追加」すればインストール完了です。

### 更新したとき

**内容を書き換えたら `sw.js` の `VERSION` を必ず上げてください。** 上げ忘れると、利用者の端末には古いままの画面が出続けます（キャッシュが更新の唯一の合図になっているため）。

Service Worker は、ある版のファイル一式をインストール時にまとめてキャッシュし、**あとから個別に書き換えません**。これは意図的な設計です。HTMLだけ新しくしてCSSを古いまま配ると、たとえば「タブを1つ増やしたHTML」と「タブ4つ前提のCSS」が噛み合って表示が壊れます。一式まるごとでしか切り替わらないようにして、この事故を防いでいます。

`VERSION` を上げると、次の起動で新しい版が裏でインストールされ、その次の起動から一式が入れ替わります（開いている画面が途中で変わることはありません）。すぐ確認したいときはアプリを完全に終了して2回開き直してください。

## ファイル構成

```
index.html              画面の骨組み
styles.css              見た目（ダークモード対応）
js/store.js             データの保存・読み書き（localStorage）
js/suggest.js           提案のスコアリング
js/scheduler.js         空き時間への割り当て（締切順）
js/ai.js                Claude API 連携（任意）
js/calendar.js          カレンダー描画
js/main.js              画面の組み立てと操作
sw.js                   オフライン用 Service Worker
manifest.webmanifest    PWA の設定
scripts/serve.mjs       確認用の簡易サーバー
scripts/make-icons.mjs  アイコンPNGの生成
```
